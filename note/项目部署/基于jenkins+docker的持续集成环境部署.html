                    # Jenkins+Docker持续集成环境部署

                    # 一、Jenkins以docker容器运行

                    # 1、前置条件

                    服务器安装docker,并且已经开放tcp端口

                    开放tcp端口，开启docker远程访问的步骤如下：

                    首先编辑docker的宿主机文件/lib/systemd/system/docker.service

                    vi /lib/systemd/system/docker.service

                    修改以ExecStart开头的行：后面追加-H tcp://0.0.0.0:2375 -H unix://var/run/docker.sock，即

                    ExecStart=/usr/bin/dockerd -H fd:// --containerd=/run/containerd/containerd.sock -H tcp://0.0.0.0:2375 -H unix://var/run/docker.sock

                    修改后保存文件，然后通知docker服务做出的修改

                    systemctl daemon-reload

                    重启docker服务

                    service docker restart

                    测试一下看是否能连接到docker api

                    curl http://localhost:2375/version

                    # 2、Jenkins安装

                    docker run -p 8080:8080 -p 50000:50000 -d -v /home/jenkins-home-docker:/var/jenkins_home jenkins/jenkins:lts

                    # 3、 Jenkins 插件安装

                    系统管理->插件管理里进行安装

                    • Docker plugin ： 提供docker 构建和发布
                    • SSH plugin ： 提供通过SSH在远程主机执行命令，用于部署服务
                    • maven plugin： 支持maven

                    # 3.1、docker配置

                    系统管理-节点管理-Configure cloud Docker Host URI 是docker主机的地址和开放的端口号

                    # 3.2、ssh主机 配置

                    “系统管理-系统设置-SSH remote hosts” 里配置SSH远程主机

                    # 3.3、配置jdk、maven

                    在系统管理-全局工具配置里设置JDK自动安装

                    # 4、 项目配置

                    # 4.1、 创建任务选择构建maven项目

                    # 4.2、 配置源码管理

                    首先配置源码，可以是git或者svn

                    # 4.3、配置触发器

                    可以配置自动构建或者手动构建

                    # 4.4、 配置Build

                    maven项目，构建使用pom.xml，执行clean package -Dmaven.test.skip=true

                    # 4.5、 配置Post Steps

                    # 4.5.1、 执行脚本

                    点击add post-build step，选择执行shell

                    # 4.5.2、 docker配置

                    点击add post-build step，选择Build / Publish Docker Image 选择cloud为先前配置的docker-cloud，image填写私服发布地址，勾上push image,构建完成后会自动push到私服

                    # 4.5.3、 远程部署

                    点击add post-build step，选择Send files or execute commands over SSH 选择配置好的主机，添加执行命令

                    docker run --name learning-test -p 8736:8080 -d registry.cn-hangzhou.aliyuncs.com/wanhx/learning:1.0
### 或者是docker compose构建部署
docker-compose -f learning.yml -p learning up -d

                    # 5、 开始构建

                    回到工程，点击立即构建，控制台会输出build success

                    # 总结

                    要想实现在Jenkins中执行docker命令，有下面三种方式： 1、将宿主机的docker文件和目录进行挂载

                    -v /var/run/docker.sock:/var/run/docker.sock   -v /usr/bin/docker:/usr/bin/docker

                    2、安装docker插件 3、直接发送命令到远程主机执行
