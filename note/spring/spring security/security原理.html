                            # 三个重要的类

                            Spring Security 的初始化和认证授权是围绕下面三类展开的：

                            DelegatingFilterProxy

                            FilterChainProxy

                            SecurityFilterChain

                            # 核心原理

                            基于容器servlet的过滤器链Filters，spring security 做其中的一个过滤器，实现认证授权的功能，在这个过滤器中管理着spring自己的过滤器链(并非 Servlet 容器中的过滤器链)。

                            两者相结合的时候通过Spring 提供了一个名为DelegatingFilterProxy的Filter实现类来实现的，DelegatingFilterProxy 就是连接二者的桥梁。在DelegatingFilterProxy下包裹这spring security提供的重要的类FilterChainProxy，这个类是spring security的功能入口，这个类下管理着许多过滤器链SecurityFilterChain，

                            # 常用的过滤器

                            # 当某个路径设置为permitAll时

                            根据过滤器调用对象FilterInvocation获取当前资源需要满足的初始化配置权限，这个权限分为两种：1、配置类中配置permitAll,2、业务控制层配置的注解权限

                            优先级： 前者大于后者，会预先获取优先级高的作为权限比较的标准，后者的注解权限则在认证通过之后访问具体资源时通过拦截器代理进行第二次动态权限认证

                            举例说明1：/user/***设置匿名访问（即permitAll），同时加上preAuthrize注解

                            在认证时用permitAll作为比较标准，认证通过，接着在访问具体资源时会再次进行认证，这次的比较标准时hasRole('ddwdada'),认证失败，匿名权限不等价与ROLE_ddwdada权限

                            举例说明1：/user/***设置匿名访问（即permitAll），方法上不加注解

                            在认证时用permitAll作为比较标准，认证通过，不会做二次的认证校验
